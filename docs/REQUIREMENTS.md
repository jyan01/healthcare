# 요구사항 정의서

## 기능 요구사항

| 기능명 | 상세 설명 |
| --- | --- |
| 회원 로그인 | 환자와 의사가 ID, Pass로 인증한 뒤 로그인한다. (회원 테이블과 데이터는 제공됨) |
| 회원 목록 조회 | 이미 등록된 회원 목록 정보를 표시 및 검색한다.<br>- 환자는 자기 자신의 정보만 조회할 수 있다.<br>- 의사는 모든 환자의 정보를 조회할 수 있다. |
| 회원 상세 조회 | 회원의 ID, 이름, 생년월일, 성별, 보유 질병 정보, 메모 정보를 조회할 수 있어야 한다.<br>실시간으로 전송되는 심박, 혈당, 걸음수 정보를 그래프로 표시한다.<br>(질병코드 테이블과 데이터는 제공됨) |
| 실시간 건강정보 조회 | 심박정보 실시간 전송 데이터: 회원ID, 측정일시, 심박수<br>혈압정보 실시간 전송 데이터: 회원ID, 측정일시, 수축기 혈압, 이완기 혈압, 혈압상태<br>체중정보 전송 데이터: 회원ID, 측정일시, 체중, BMI<br>걸음수 실시간 전송 데이터: 회원ID, 전송일시, 당일 누적 걸음수<br>건강정보 제공 서버 있음 (웹소켓으로 데이터 제공, https://healthsim.iranglab.com) |
| 알림 기능 | 심박, 혈압, 혈당 정보 수신 중 이상 데이터 감지 시 관리자에게 알림을 전송한다. |
