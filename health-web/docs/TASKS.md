# TASKS

## 작업할 내용

## 완료된 작업

- 로그인 웹용화면
- 회원목록 웹용화면
- 회원상세 웹용화면

### 고도화

- AI 상담(챗봇) 화면 (`/chat`) — health-ai RAG Agent 연동
- 회원상세 화면 "AI 소견 요약" 버튼 (최근 데이터+보유질환 기반 AI 요약)
- 회원상세 화면 기간별 조회(startAt~endAt) 기능
- (health-backend) 이상감지 Slack 알림 문구 AI 생성 + 실패 시 기본 문구 fallback
- 회원목록 화면에 최근 24시간 내 이상감지 표시(`hasRecentAlert`) — 의사가 목록에서 바로 주의 필요 환자 확인 가능. 그 환자 상세화면을 열어서 확인하면(`last_alert_ack_at` 갱신) 빨간 점이 사라지고, 이후 새로 발생한 이상감지에만 다시 표시됨
- 수면 데이터 신규 지원 — 시뮬레이터가 이미 보내주지만 백엔드가 전혀 저장하지 않던 `sleep` 이벤트를 엔티티/DB테이블/WS 브로드캐스트까지 새로 추가하고, 회원상세 화면에 수면 요약 카드로 표시 (하루 1회만 수신되는 값이라 다른 지표와 달리 스트리밍 차트 대신 요약 카드로 표시)

### 기본 요구사항 보완

- 회원상세 화면에 보유 질병 정보(칩)와 진단 메모(`diagContent`)를 표시 (`docs/REQUIREMENTS.md`의 "보유 질병 정보, 메모 정보" 요구사항 — 데이터는 API에 있었지만 화면에 렌더링되지 않던 것을 보완)
- 회원상세 화면 실시간 건강정보 그래프에 걸음수(stepCount) 카드 추가 (`docs/REQUIREMENTS.md`의 "실시간 건강정보 조회" 요구사항 — WebSocket/REST 배관은 이미 있었지만 화면에 표시되지 않던 것을 보완)
